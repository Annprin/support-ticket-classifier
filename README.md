# Проект: "Telco Churn Predictor" (Нейросеть для прогнозирования оттока)
Этот проект — ML-сервис для прогнозирования вероятности ухода клиента (оттока) телеком-компании. Модель построена на базе нейронной сети (MLP) и полностью интегрирована в экосистему Hugging Face (transformers).

1. **Цель проекта**
Разработка API-сервиса, который по набору признаков клиента (тариф, срок контракта, платежи) возвращает вероятность его ухода.

**Бизнес-задачи:**

* Идентификация "группы риска": Выявить клиентов с вероятностью оттока > N%.

* Таргетированное удержание: Позволить отделу маркетинга сосредоточить усилия (скидки, спецпредложения) только на группе риска.

* Оптимизация бюджета: Снизить расходы на удержание, не предлагая скидки лояльным клиентам.

2. **Набор данных (Dataset)**
* Датасет: "Telco Customer Churn" с Kaggle.

* Состав: ~7000 клиентов, ~20 признаков (категориальные и числовые).

* Признаки: gender, SeniorCitizen, tenure (срок "жизни"), InternetService, MonthlyCharges (мес. платеж) и т.д.

* Целевая переменная (Target): Churn (Yes/No).

* Особенность: Датасет требует предобработки (One-Hot Encoding, Scaling, обработка пропусков), которая будет "зашита" в пайплайн.

3. **План экспериментов**

Цель: Выполнить требования проекта и добиться F1-score ≥0.60.

Модель: Кастомный Multi-Layer Perceptron (MLP), написанный на PyTorch.

Архитектура (из config.yaml):

Input (зависит от кол-ва признаков после OHE)

Hidden 1: 64 нейрона + ReLU + Dropout

Hidden 2: 32 нейрона + ReLU

Output: 2 нейрона (Логиты для классов "No" и "Yes")

Интеграция: Модель будет написана как кастомный класс, наследуемый от transformers.PreTrainedModel, что позволит использовать model.save_pretrained() и model.from_pretrained().
